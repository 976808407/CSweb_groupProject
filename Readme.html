Q & A

  • 记录项目开发过程中遇到的问题和解决方案

  • Q:项目开发过程中 各成员代码重复部分很多 导致代码冗余

    A:对于重复的功能模块要优先封装然后共享到组内 尽可能减少重复工作

  • Q:各成员变量及服务端接口命名冲突

    A:给每个成员的变量和接口加前缀以区分

  • Q:单个模块功能实现 但合并项目时功能逻辑出错

    A:合并项目过程中填补逻辑缺陷

  • Q:原本实现的功能 合并后不报错也不执行

    A:统计好各成员所引用的所有组件及环境 同时注意兼容性差异

  • ...

经验总结

1.做好项目分析，让每个成员对项目都有一个细致了解，防止出现功能逻辑缺陷

2.优先封装多个成员都需要使用的功能模块共享到组内，避免大量重复工作

3.规范变量 函数 接口的命名，防止项目合并时出现冲突

4.定期开会 汇报进度以及开发中的问题 避免重复踩雷
